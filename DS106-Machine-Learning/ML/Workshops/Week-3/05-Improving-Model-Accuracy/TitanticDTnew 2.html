How about our "age" column? Should we just drop the missing values and lose 20% of our data? Or are there other methods of handling missing values?

When creating machine learning models, "imputing missing values" is high priority. More data means better models.

Skipping to a few steps forward, here is the results of a decesion tree using the data with missing values in the "age" column just brutely dropped.

Can we get a better accuracy score than 77% by imputing missing values rather than just dumping rows with NaN's? Well lets give it a shot!

One of the simple approaches for handling missing values for continuous variables is replacing missing values with the mean or median of the variable. We will go one step further and fill the missing values in the "age" column with the average age for each economic class. Perhaps the distribution of age was different for the first, second, and third class?

In [9]:
titanic.groupby('pclass').mean()['age']
Out[9]:

Looks like the average age was significatnly higher for first class. Lets impute the missing values in the "age" column with values above. Its a doosy of a line, but it is handy none the less!

In [10]:
titanic["age"] = titanic.groupby("pclass").transform(lambda x: x.fillna(x.mean()))['age']

Now that we have dealt with our NaN's, lets recode our catergorical variables. All predictor variables (our x varibles) passed into any Sci-kit Learn (sklearn) algorithm (classifier or regressor) need to be either integers or floats. Catergorical variables:

  • If binary, the groups need to be changed to 0 or 1
  • If ordinal, they should numbered in a discrete sequence (i.e. "pclass")
  • If non-ordered catergory groups, they will be dummy coded

Tip! You do not need to recode variables of the boolean type, because they are already processed as integer 0 (False) or integer1 (True). How neat!

titanic['sex'] = titanic['sex'].replace(['male', 'female'], [0,1])
